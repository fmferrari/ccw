# Release 0.1.20

## Summary

`ccw-mcp` `0.1.20` implements Phase 5L docs-lane subject-pairing after downstream
feedback on `0.1.19`. Implementation/test lanes and agentic anchors were already
acceptable, but the docs lane could still lead with broad Hermes/runtime docs for
a retrieval-ranking documentation task.

This patch keeps ranking deterministic and generic: no LLM reranker, vector store,
semantic-search service, repo-specific path rules, or wikiagent authority was
added to CCW core.

## What changed

- **Primary subject coupling for docs.** Docs-mode subject coupling now uses
  primary task subject terms instead of alias-expanded terms. Aliases remain
  secondary topical hints, but broad `search`/`query` mentions no longer make a
  generic doc lead a retrieval-ranking docs task.
- **Stricter multi-subject docs gate.** Multi-subject prompts such as
  `retrieval` + `ranking` require stronger subject coupling before a
  documentation-shaped file can lead.
- **Precise docs adjacency exemption.** Deterministic adjacency can still help
  docs rank, but only precise anchor/path/basename evidence can exempt a doc from
  broad-doc topicality penalties.
- **Richer capped symbol scoring.** Source/test symbols can score multiple
  matched subject terms while preserving the existing cap, so retrieval-ranking
  anchors are not underweighted against broad docs.

## Regression coverage added

- `tests/test_compile.py` adds
  `test_rank_file_lanes_docs_mode_requires_specific_subject_pairing_for_broad_docs`,
  covering the downstream audit shape where broad runtime docs mention generic
  retrieval/search terms but should not outrank retrieval/ranking source and tests.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.20 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.20`, and the path must not point to a stale vendored copy.
