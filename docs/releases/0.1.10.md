# Release 0.1.10

## Summary

`ccw-mcp` `0.1.10` refines lane-quality ranking for the latest downstream audit:
it filters build-artifact noise at ranking time and reduces cross-language
frontend leakage in backend-oriented retrieval/refactor tasks.

## What changed

- **Ranking-time build artifact filtering.** Files under `build/`, `dist/`, and
  `dev-dist/` are now excluded during lane ranking even if they exist in an
  older index snapshot.
- **Task-language affinity for code-heavy tasks.** Ranking now infers a
  preferred task language from path and symbol signals and applies
  cross-language penalties for `bugfix`/`implementation`/`refactor` tasks.
- **Frontend leakage reduction for backend tasks.** Added stronger penalties
  for frontend tree files (`app/`, `frontend/`, `pages/`, etc.) when the task
  text does not contain frontend-oriented hints.

## Regression coverage added

- Added deterministic tests in `tests/test_compile.py` to ensure:
  - frontend TypeScript files do not outrank retrieval Python files for
    backend refactor tasks.
  - build artifact paths are excluded from both task and agentic lanes.

## Validation

- `.venv/bin/python -m unittest` -> `177 tests` passed.
- `.venv/bin/python -m unittest tests.test_cli_classify tests.test_recipes tests.test_mcp_server tests.test_compile tests.test_cli_index` -> `98 tests` passed.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.10 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.10` and path must not point to a vendored module copy.
