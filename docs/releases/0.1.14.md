# Release 0.1.14

## Summary

`ccw-mcp` `0.1.14` is a patch release for the lane-quality audit follow-up.
It keeps docs-shaped tasks from displacing the canonical repo anchors out of the
agentic lane, restores refactor classification for retrieval-ranking cleanup
tasks phrased as "clarity"/"preserving behavior", and reduces duplicate
instruction-surface crowding in `## Agentic Context`.

## What changed

- **Pinned core agentic anchors.** `AGENTS.md`, `wiki/AGENTS.md`, `CONTEXT.md`,
  `wiki/user/index.md`, and `wiki/user/log.md` now stay in `## Agentic Context`
  even for docs-shaped tasks.
- **Stronger refactor keywords.** Added `clarify`, `clarity`, `preserve`, and
  `preserving` to the deterministic `refactor` classifier keywords so prompts
  like "Refactor retrieval ranking flow for clarity while preserving behavior"
  no longer classify as `docs`.
- **Instruction-family capping.** Repeated `.github`/`.opencode`/`.cursor`
  instruction files are capped to one per family in the agentic lane so
  duplicate harness guidance does not crowd out canonical anchors.
- **Regression coverage.** Added tests for refactor classification, docs-mode
  anchor pinning, and instruction-family capping.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.14 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the editor or harness process after upgrading.

## Post-update verification

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.14`, and the path must not point to a vendored module copy.
