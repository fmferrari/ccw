# Release 0.1.21

## Summary

Superseded by `0.1.22`, which includes the final top-five visibility
recombination fix. Use `0.1.22` or newer for Phase 5M validation.

`ccw-mcp` `0.1.21` implements Phase 5M docs-lane evidence composition after
downstream feedback on `0.1.20`. It keeps topicality-first docs ranking while
ensuring docs-shaped tasks include a human-authored documentation destination
when no qualified doc can lead.

This patch stays deterministic and generic: no LLM reranker, vector store,
semantic-search service, repo-specific path rules, public artifact schema change,
or wikiagent authority was added to CCW core.

## What changed

- **Docs-mode lane packing.** Ranking still lets qualified docs or strong
  behavior evidence lead, then assembles the docs task lane so a qualified doc
  below the top five can be promoted into view.
- **Fallback doc destination.** If no qualified doc exists and the task lane has
  at least five slots, CCW inserts the best non-clutter human-authored doc
  destination at slot five while preserving the top three behavior-evidence
  files.
- **Clutter exclusion.** Generic indexes/logs, root config/lock clutter,
  generated/build artifacts, vendored/tool copies, and agent instruction files
  are not inserted as fallback doc destinations.
- **Fixture semantics.** JSON/YAML fixtures remain behavior evidence by default;
  they can rank by relevance but do not satisfy the fallback-doc slot unless the
  task explicitly asks to document fixtures, schemas, or structured examples.

## Regression coverage added

- `tests/test_compile.py` adds docs-mode regressions for:
  - inserting `docs/README.md` into the top five when the lane would otherwise be
    pure source/test/benchmark/fixture evidence.
  - excluding generic spec indexes from fallback insertion.
  - treating topical JSON fixtures as behavior evidence rather than fallback docs.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.21 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.21`, and the path must not point to a stale vendored copy.
