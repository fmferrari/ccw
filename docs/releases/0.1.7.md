# Release 0.1.7

## Summary

`ccw-mcp` `0.1.7` improves docs-intent lane ranking so architecture/spec
documentation under wiki/docs structures is prioritized as task evidence for
documentation-shaped tasks.

## What changed

- **Architecture/spec documentation now ranks as first-class docs evidence.**
  For docs-intent tasks, task-lane boosts now prioritize generic documentation
  structures such as `architecture/`, `design/`, `adr/`, `spec/`, `specs/`,
  `runbook/`, and `guide/`.
- **Spec paths are no longer misclassified as test trees for markdown docs.**
  `spec`/`specs` directory hints now behave as test-path hints only for code
  files; markdown documentation under those paths remains docs evidence.
- **Docs-intent ranking remains generic.** Heuristics are path-shape and
  extension based; no repository-specific hardcoded path boosts were added.

## Regression tests added

- Added deterministic coverage that docs-shaped tasks prioritize wiki
  architecture/spec pages in the task lane while keeping `AGENTS.md` in the
  agentic lane (`tests/test_compile.py`).

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.7 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

## Post-update verification

1. Confirm the running source and version:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

2. Restart the editor/harness process that hosts the MCP server.
3. Re-index and compile a docs-shaped artifact:

```bash
ccw index .
ccw compile --task "Document retrieval ranking behavior and troubleshooting notes" --out .ccw/compiled/lane-check.md
rg "^## " .ccw/compiled/lane-check.md
```
