# Release 0.1.12

## Summary

`ccw-mcp` `0.1.12` fixes two lane-quality issues from the 0.1.10 audit:
docs-shaped queries now correctly classify as `docs` mode instead of falling
through to `implementation`, and retrieval-ranking source files get stronger
path matching for refactor tasks.

## What changed

- **Broader docs classification keywords.** Added `"behavior"` and `"behaviour"`
  to the `docs` mode keyword set so tasks like "Document retrieval ranking
  behavior" classify as docs mode instead of implementation.
- **Bidirectional keyword aliases.** Added `"document"` → `{doc, wiki, spec, guide}`,
  `"documentation"` → `{doc, wiki, spec, guide}`, `"search"` ↔ `{retrieval, query,
  lookup}`, and `"rank"` → `{score, order, sort, tie}` so wiki/doc files and
  retrieval-ranking modules like `scripts/wiki_search.py` score higher in base
  path matching for the relevant task types.

## Validation

- `python -m pytest tests/` → `177 tests` passed.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.12 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.12` and path must not point to a vendored module copy.
