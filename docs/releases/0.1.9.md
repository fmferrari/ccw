# Release 0.1.9

## Summary

`ccw-mcp` `0.1.9` addresses the latest lane-quality audit by introducing an
explicit docs mode and further tuning refactor/docs ranking behavior to better
surface task-relevant files.

## What changed

- **New deterministic `docs` classification mode and recipe.** Documentation
  tasks now classify to `docs` instead of falling back to `implementation`,
  with a recipe tuned for doc evidence.
- **Mode-aware lane ranking.** The ranker now receives task mode context from
  the compiler pipeline so docs/refactor heuristics can be applied precisely.
- **Stronger docs-lane prioritization for docs tasks.** Docs candidates under
  wiki/docs/spec-style paths now get higher boosts and are prioritized into the
  task lane under docs mode.
- **Refactor test-file dominance reduced.** Refactor mode now applies stronger
  penalties to test files while giving additional source/code preference.
- **Symbol inflation guard.** Per-file symbol-name match boosts are now capped
  to prevent large test files from winning solely by symbol volume.
- **Retrieval semantic relevance expanded.** Added alias terms such as
  planner/router/tie/sort to improve retrieval-ranking file alignment.

## Regression coverage added

- `tests/test_cli_classify.py`:
  - docs-shaped prompt classifies to `docs`.
- `tests/test_recipes.py`:
  - docs recipe lookup, budgets, and allocation invariants.
- `tests/test_mcp_server.py`:
  - MCP classify endpoint returns `docs` for docs-shaped prompts.
- `tests/test_compile.py`:
  - refactor lane de-prioritizes tests under dense symbol overlap.
  - docs mode forces documentation files to lead the task lane.

## Validation

- `.venv/bin/python -m unittest` -> `175 tests` passed.
