# Release 0.1.24

## Summary

`ccw-mcp` `0.1.24` adds `wiki/`, `docs/`, and `doc/` to the docs-mode priority
path segments, fixing a gap identified by the downstream wikiagent lane audit.
Files under these canonical documentation directories now receive the same
+10 priority boost in docs mode that `architecture/`, `spec/`, `guide/`, and
`adr/` already enjoyed.

## What changed

- **Wiki/docs paths now get docs-mode priority boost.** `_TASK_DOC_PRIORITY_PATH_SEGMENTS`
  gained `"wiki"`, `"docs"`, and `"doc"`. Any file under a `wiki/`, `docs/`, or `doc/`
  directory now receives the +10 priority boost in docs mode, on top of the base +20
  doc-candidate boost.
- **No behavioral regression.** The topicality gate (-55 penalty) still prevents
  unrelated doc candidates from displacing strong source/test matches. All 194 unit
  tests pass, including the 20+ docs-mode-specific ranking tests.

## Background

The wikiagent lane audit flagged that for a "Document retrieval ranking behavior"
docs-mode task, the top 5 ranked files were all source/test/fixture. A fallback
doc destination (`wiki/user/ops/plans/harness-runtime-migration-plan.md`) existed at
position #7 but received only the base +20 boost, missing the extra +10 priority
boost because `"wiki"` wasn't in the priority path segments. The fix surfaces
wiki/docs files naturally in docs mode without changing the fallback promotion
mechanics.

## Validation

- `python -m unittest tests.test_compile.RankFilesTests -q` -> passed.
- `python -m unittest -q` -> `194 tests` passed.
- `python -m build --sdist --wheel && python -m twine check dist/*` -> passed.
- Published with `scripts/release-pypi.sh`; PyPI JSON verified version `0.1.24`
  with wheel and sdist artifacts.
- Clean venv install of `ccw-mcp==0.1.24` printed runtime version `0.1.24`.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.24 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.24`, and the path must not point to a stale vendored copy.
