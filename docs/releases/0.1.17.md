# Release 0.1.17

## Summary

`ccw-mcp` `0.1.17` tunes docs-mode lane ranking after the downstream
`wikiagent` audit of `0.1.16`. Implementation-shaped and test-shaped tasks were
acceptable, but the docs-shaped task still led with benchmark/source files before
a wiki/spec document.

This patch keeps ranking deterministic and generic: no LLM reranker, vector
store, semantic-search service, or repo-specific path rules were added.

## What changed

- **Docs-intent topicality for docs candidates.** Documentation-shaped files can
  now earn topicality from docs-intent terms such as behavior, troubleshooting,
  notes, guide, and README.
- **Source/test behavior preserved.** Those same docs-intent terms remain weak
  for source/test ranking, so implementation and test lanes do not broaden.
- **Docs-lane gate moderation.** A moderately topical wiki/spec/doc candidate can
  clear the docs topicality gate and lead before benchmark/source files when no
  stronger retrieval-specific docs are indexed.

## Regression coverage added

- Added a deterministic regression in `tests/test_compile.py` for the
  `wikiagent` docs-shaped audit task: `Document retrieval ranking behavior and
  troubleshooting notes`.

## Validation

- `python -m unittest tests.test_compile.RankFilesTests` -> `33 tests` passed.
- `python -m unittest -q` -> `186 tests` passed.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.17 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.17`, and the path must not point to a stale vendored copy.
