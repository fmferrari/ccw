# Release 0.1.8

## Summary

`ccw-mcp` `0.1.8` hardens lane-quality ranking based on the latest downstream
audit: docs-shaped tasks now keep documentation evidence in the task lane under
high lexical competition, and refactor tasks better prioritize source over
tests.

## What changed

- **Docs-intent lane now wins against dense retrieval overlap.** Increased
  docs-intent boosts ensure wiki/docs/spec artifacts stay in `## Files` for
  docs-shaped prompts, even when many test/source files share retrieval/ranking
  terms.
- **Refactor intent now prefers source and de-emphasizes tests.** Added
  refactor-hint detection and stronger role weighting so
  "refactor ... preserving behavior" tasks rank implementation files ahead of
  regression tests.
- **Retrieval/ranking semantic matching improved.** Task-term aliases
  (retrieval/search/query/lookup and ranking/rank/score/order) plus
  underscore-aware token matching improve relevance for files like
  `scripts/wiki_search.py`.

## Regression tests added

- Added deterministic coverage for refactor-task source prioritization over
  tests (`tests/test_compile.py`).
- Added deterministic coverage for retrieval/ranking alias-term semantic
  matching (`tests/test_compile.py`).
- Added deterministic coverage that docs-intent tasks prioritize wiki
  architecture/spec documentation even when retrieval-related tests/source files
  have high overlap (`tests/test_compile.py`).

## Validation

- `.venv/bin/python -m unittest` -> `171 tests` passed.
- `python -m unittest tests.test_cli_index tests.test_compile` -> `52 tests`
  passed.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.8 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

## Post-update verification

1. Confirm running version and source path:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

2. Version must be `0.1.8`; path must not be a vendored module copy.
3. Restart the harness/editor process hosting the MCP server.
