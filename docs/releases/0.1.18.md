# Release 0.1.18

## Summary

`ccw-mcp` `0.1.18` implements Phase 5J subject-coupled docs ranking and
deterministic wiki-structure evidence. Docs-shaped tasks no longer let generic
behavior/spec pages lead solely because they look like documentation; docs need
subject coupling or deterministic adjacency to strong code/test anchors.

This patch keeps the critical path deterministic and generic: no LLM reranker,
vector store, semantic-search service, repo-specific path rules, or wikiagent
authority was added to CCW core.

## What changed

- **Subject-coupled docs gate.** Docs candidates can lead docs-mode lanes when
  they match task subject terms such as retrieval/ranking/tie/benchmark, not from
  docs-intent terms alone.
- **Deterministic docs adjacency.** Docs can earn an adjacency boost when indexed
  evidence links or mentions strong task-relevant code/test anchors.
- **Wiki-structure evidence.** Markdown frontmatter title/tags/type/status,
  wikilinks, and markdown links are exposed in artifact search text for ranking.
- **Near-zero clutter suppression.** Generic root config, lockfiles, Dockerfiles,
  and OS junk are suppressed from code-task lanes unless the task asks for config,
  infra, build, dependency, harness, or tooling work.

## Regression coverage added

- Docs subject coupling prevents generic behavior/spec pages from leading before
  topical source/tests.
- Frontmatter/link-backed docs can lead via deterministic adjacency.
- Generic clutter stays out of implementation/refactor task lanes.
- Markdown artifact indexing exposes frontmatter, wikilinks, and markdown links.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.18 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.18`, and the path must not point to a stale vendored copy.
