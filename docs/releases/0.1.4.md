# Release 0.1.4

## Summary

`ccw-mcp` `0.1.4` hardens lane ranking with generic, repository-agnostic
heuristics so outputs stay valid across different project layouts and harness
ecosystems.

## What changed

- Removed repository-specific lane boosts for fixed paths under `wiki/user/...`.
- Added generic anchor detection for agentic context files:
  - anchor basenames such as `AGENTS.md` and `CONTEXT.md`
  - structure files such as `index.md`, `log.md`, and `changelog.md` when
    they live under common knowledge roots (`docs`, `wiki`, `knowledge`,
    `notes`, `handbook`, `playbook`).
- Prioritized anchor files ahead of other agentic context candidates.
- Kept third-party and vendor trees as fallback-only in the task lane so
  `vendor/`, `third_party/`, `node_modules/`, and similar paths do not crowd
  first-party implementation evidence.
- Added task-intent-aware file weighting:
  - implementation-oriented tasks prefer source/code trees
  - explicit test tasks prefer test files
  - explicit docs tasks prefer docs/spec/wiki files.
- Added regression tests for:
  - anchor prioritization in the agentic lane
  - third-party fallback ordering in the task lane
  - implementation-vs-test intent prioritization.
- Added a reusable wikiagent lane-audit prompt that includes explicit
  post-release client update instructions.

## Why this release matters

Lane behavior now generalizes better across heterogeneous repositories. The
compiler no longer assumes one documentation topology while still preserving
strong defaults for agentic anchors, implementation evidence, and vendor
suppression.

## Client update guide

Update your installation according to how your client launches CCW.

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.4 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

If you target multiple harnesses, run install per target as needed.

## Post-update verification

1. Restart the editor/harness process that hosts the MCP server.
2. Re-index the target repo:

```bash
ccw index .
```

3. Compile a fresh artifact and confirm lane sections:

```bash
ccw compile --task "Verify lane behavior" --out .ccw/compiled/lane-check.md
rg "^## " .ccw/compiled/lane-check.md
```

Expected section headers include:

- `## Files`
- `## Agentic Context` (when qualifying context files are present)

## Wikiagent audit prompt

Use `docs/prompts/wikiagent-lane-regression.md` for an end-to-end lane audit.
It includes a required step to upgrade wikiagent clients after a new CCW
release and restart the harness process.
