# Release 0.1.13

## Summary

`ccw-mcp` `0.1.13` is a corrective patch release that aligns all public version
metadata after the `0.1.12` PyPI artifacts were published with stale `0.1.11`
values in the package runtime and APM manifest.

This release includes the `0.1.12` behavior fixes for docs classification and
retrieval-ranking aliases. It adds no new compiler behavior.

## What changed

- Updated `pyproject.toml`, `src/ccw/__init__.py`, and `apm.yml` to `0.1.13`.
- Updated README and downstream audit upgrade pins to `0.1.13`.
- Added this release note documenting why `0.1.13` supersedes `0.1.12`.

## Validation

- `.venv/bin/python -m unittest`
- `.venv/bin/python -m build`
- `.venv/bin/python -m twine check dist/*`

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.13 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the editor or harness process after upgrading.

## Post-update verification

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.13`, and the path must not point to a vendored module copy.
