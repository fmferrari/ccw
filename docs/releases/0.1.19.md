# Release 0.1.19

## Summary

`ccw-mcp` `0.1.19` implements Phase 5K lane-audit tightening after downstream
feedback on `0.1.18`. It keeps the Phase 5J subject-coupled docs model, while
closing residual edge cases around regression-test classification and root config
noise.

This patch remains deterministic and generic: no LLM reranker, vector store,
semantic-search service, repo-specific path rules, or wikiagent authority was
added to CCW core.

## What changed

- **Regression-test prompt classification.** Prompts combining
  regression/stability/coverage wording with test/tests now classify as `review`,
  which uses the test/review-biased recipe instead of broad implementation mode.
- **Root config clutter suppression.** Generic root JSON/YAML/TOML/lock config
  files are suppressed from task lanes unless the task explicitly asks for
  config, infra, build, dependency, harness, release, or tooling work.
- **Docs-lane semantic relevance regression.** Added coverage for the downstream
  audit shape where unrelated docs-shaped wiki/spec/notes pages must not outrank
  subject-relevant retrieval/ranking source or tests when no topical docs exist.

## Regression coverage added

- `tests/test_cli_classify.py` asserts the retrieval-ranking regression-test
  prompt classifies as `review`.
- `tests/test_compile.py` asserts subject-relevant retrieval source outranks
  unrelated docs-shaped files and root JSON clutter for the docs audit shape.
