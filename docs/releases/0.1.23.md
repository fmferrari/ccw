# Release 0.1.23

## Summary

`ccw-mcp` `0.1.23` is the Phase 5O docs-lane leading composition release after
the `0.1.22` feedback cycle. It keeps docs ranking deterministic while making
eligible documentation destinations lead when the task is genuinely
docs-shaped.

The release preserves the existing Phase 5M fallback behavior for generic docs
destinations, but it no longer lets unrelated docs displace topical
source/test/benchmark evidence.

## What changed

- **Qualified docs can lead.** Up to two sufficiently topical documentation
  candidates can now occupy the leading task-lane slots in docs mode.
- **Subject-relevant fallback promotion.** When no qualified doc exists, the
  lane assembler promotes one weaker but subject-relevant human-authored doc to
  slot one.
- **Unrelated docs stay supportive.** When docs candidates are unrelated to the
  task subject, source and test evidence keep the lead.
- **Phase 5M support behavior remains.** Generic fallback doc destinations can
  still satisfy the top-five docs-mode requirement without forcing unrelated
  docs to the front.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.23 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.23`, and the path must not point to a stale vendored copy.
