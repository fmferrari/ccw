# Release 0.1.5

## Summary

`ccw-mcp` `0.1.5` is a correctness follow-up to the `0.1.4` lane-quality work.
It fixes three lane-ranking defects surfaced by a downstream regression audit
that persisted even when `0.1.4` was running.

## What changed

- **Agentic anchors survive tighter lane limits.** Under the implementation
  recipe the default agentic slot heuristic only allocated four slots, so the
  fifth anchor (commonly `wiki/user/log.md`) was evicted by an alphabetical
  tiebreak. The agentic lane now grows to fit every anchor, bounded so it never
  starves the task lane on small budgets.
- **Documentation intent is detected more broadly.** Tasks phrased as
  "document …" or "documentation" now trigger docs-intent weighting. The doc
  hint vocabulary added `document`, `documentation`, `explain`, `note`,
  `notes`, `manual`, `tutorial`, `comment`, `comments`, and `troubleshooting`.
- **Snippet extraction no longer overshoots the budget.** Once a lane's token
  budget was exhausted, later files were emitted in full because the truncation
  guard only fired while budget remained positive. Files past the budget now
  keep their reference but drop the snippet body.

## Regression tests added

- All agentic anchors stay in the agentic lane under the default item limit.
- A "document …" task ranks docs/spec files ahead of source and tests.
- `extract_snippets` never exceeds its budget across multiple files.

## Note on the 0.1.4 audit

The audit that motivated this release reported "no behavioral change between
0.1.3 and 0.1.4". That signal indicated the MCP server was still running an
older binary (for example a vendored copy or a cached `uvx` launcher) rather
than the upgraded package. Before re-auditing, confirm the running source:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

If `getfile(ccw)` does not point at the upgraded install, repoint the MCP
launch (or re-vendor) and restart the harness. Without that, no CCW release
will take effect.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.5 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

## Post-update verification

1. Confirm the running source and version:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

2. Restart the editor/harness process that hosts the MCP server.
3. Re-index and compile a fresh artifact:

```bash
ccw index .
ccw compile --task "Document retrieval ranking behavior" --out .ccw/compiled/lane-check.md
rg "^## " .ccw/compiled/lane-check.md
```

## Wikiagent audit prompt

The end-to-end lane audit prompt lives in the maintainer's private wiki at

private wiki is mounted). It now runs client upgrade plus a running-binary
