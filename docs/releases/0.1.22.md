# Release 0.1.22

## Summary

`ccw-mcp` `0.1.22` is the corrective Phase 5M docs-lane evidence composition
release. It supersedes `0.1.21`, which was published before the final top-five
visibility recombination fix.

The release keeps docs-mode ranking topicality-first while ensuring a docs task
includes a human-authored doc destination in the first five task-lane items when
no qualified doc can lead.

## What changed

- **Top-five composition fix.** Fallback doc insertion now checks the first five
  task-lane items, not the whole larger task lane. A fallback doc already ranked
  sixth or lower is promoted into slot five.
- **Correct promotion helper.** The lane assembler now promotes candidates that
  are already inside the larger task lane but outside the top-five visibility
  window.
- **Stronger regression.** The fallback-doc test now uses a larger task lane so
  it catches the recombination bug where a doc destination existed but was not in
  the top five.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.22 --help
```

### APM-managed clients

```bash
apm install fmferrari/ccw --target copilot
```

Restart the harness/editor process after upgrade, then verify:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

Version must be `0.1.22`, and the path must not point to a stale vendored copy.
