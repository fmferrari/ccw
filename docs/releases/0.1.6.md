# Release 0.1.6

## Summary

`ccw-mcp` `0.1.6` ships the Phase 5F lane-quality fixes from the 2026-06-23
audit: documentation-intent task lanes now lead with documentation evidence,
and build-artifact directories no longer pollute indexed task lanes.

## What changed

- **Documentation-intent task-lane boost.** The ranking pass now detects
  docs-shaped intent (for example: "document", "documentation",
  "troubleshooting", "guide", "readme", "spec") and deterministically boosts
  generic documentation-shaped files in the task lane:
  - markdown/reStructuredText extensions (`.md`, `.rst`, related doc formats)
  - generic doc roots (`docs/`, `wiki/`, `notes/`, etc.)
  - spec-named documentation files
- **Docs stay in task evidence lane for docs tasks.** Documentation-shaped files
  can now appear in the task lane even when they might also look like
  agentic-context candidates; harness/context anchors (`AGENTS.md`,
  `CONTEXT.md`, MCP/Copilot instruction/config files) remain in the agentic
  lane.
- **Build-artifact exclusion expanded.** Indexing now excludes `dev-dist/`
  alongside existing `dist/` and `build/` exclusions, preventing generated
  bundles from occupying high-value ranking slots.

## Regression tests added

- Docs-shaped tasks lead task-lane output with docs/spec files while preserving
  `AGENTS.md` in the agentic lane (`tests/test_compile.py`).
- Build-artifact directories (`dev-dist`, `dist`, `build`) are excluded from
  indexed file inventory (`tests/test_cli_index.py`).

## Remaining follow-up

The retrieval benchmark fixture refresh (`retrieval_vs_analysis` `planner_state`
alignment) is a downstream fixture-maintenance task in the sibling checkout
that contains those benchmark assets. This release keeps ranking/index behavior
changes isolated to CCW core.

## Client update guide

### pip

```bash
pip install -U ccw-mcp
```

### pipx

```bash
pipx upgrade ccw-mcp
```

### uvx launchers

```bash
uvx --refresh ccw-mcp==0.1.6 --help
```

### APM-managed MCP clients

```bash
apm install fmferrari/ccw --target copilot
```

## Post-update verification

1. Confirm the running source and version:

```bash
python -c "import ccw, inspect; print(ccw.__version__, inspect.getfile(ccw))"
```

2. Restart the editor/harness process that hosts the MCP server.
3. Re-index and compile a docs-shaped artifact:

```bash
ccw index .
ccw compile --task "Document retrieval ranking behavior and troubleshooting notes" --out .ccw/compiled/lane-check.md
rg "^## " .ccw/compiled/lane-check.md
```
